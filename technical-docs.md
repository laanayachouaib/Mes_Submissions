# Documentation Technique

## Architecture
Le projet suit une architecture modulaire.

### Structure
- `/docs` : Documentation
- `/src` : Code source
- `/tests` : Tests unitaires

## Technologies utilisées
- Git pour le version control
- Markdown pour la documentation
- Shell pour l'automatisation

## Spécifications fonctionnelles

### Fonctionnalités principales
1. Gestion du version control
2. Documentation automatique
3. Workflow de validation

### Critères d'acceptation
- Tous les commits doivent avoir des messages clairs
- Les branches doivent être nommées selon les conventions
- Les merges doivent être validés

## Bonnes pratiques

### Conventions de commits
- **feat**: Nouvelle fonctionnalité
- **fix**: Correction de bug
- **docs**: Documentation
- **refactor**: Refactorisation
- **test**: Tests
- **chore**: Maintenance

### Workflow Git
1. Créer une branche depuis main
2. Faire des petits commits atomiques
3. Faire une revue de code
4. Squasher les commits avant merge
5. Fusionner avec un message clair"
