# Configuration Git - user.name et user.email

## Commandes utilisées pour configurer l'identité Git

### Configuration locale (pour ce projet uniquement)
git config user.name "Votre Nom"
git config user.email "votre.email@exemple.com"

### Configuration globale (pour tous les projets)
git config --global user.name "Votre Nom"
git config --global user.email "votre.email@exemple.com"

### Vérifier la configuration
git config user.name
git config user.email
git config --list

## Différence entre configuration locale et globale

| Type | Portée | Commande | Fichier de configuration |
|------|--------|----------|-------------------------|
| Locale | Projet actuel uniquement | `git config` | .git/config |
| Globale | Tous les projets de l'utilisateur | `git config --global` | ~/.gitconfig |

## Pourquoi configurer user.name et user.email ?

Ces informations sont essentielles car elles sont associées à chaque commit que vous créez. Elles permettent de :
- Identifier l'auteur de chaque modification
- Assurer la traçabilité des changements
- Collaborer efficacement en équipe
- Afficher correctement l'historique des commits

## Vérification après configuration
git log --oneline --pretty=format:"%h %an %ae %s"
